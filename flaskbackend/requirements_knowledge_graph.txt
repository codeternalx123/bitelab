# ===========================
# Version Compatibility Notes
# ===========================
#
# Python Version: 3.9+ required
# - Some ML libraries require newer Python versions
# - Async features require Python 3.7+
# - Type hints work best with Python 3.9+
#
# Operating System Compatibility:
# - Windows: Full support with Visual C++ Build Tools
# - Linux: Full support (Ubuntu 20.04+, CentOS 8+)
# - macOS: Full support (10.15+, both Intel and Apple Silicon)
#
# Hardware Requirements:
# - RAM: 8GB minimum, 16GB recommended
# - CPU: Multi-core recommended for ML operations
# - GPU: Optional but recommended for large ML models
# - Storage: 10GB minimum for models and cache
#
# ===========================
# Installation Commands
# ===========================
#
# Standard Installation:
# pip install -r requirements_knowledge_graph.txt
#
# Development Installation (includes dev tools):
# pip install -r requirements_knowledge_graph.txt
#
# Production Installation (minimal):
# pip install --no-dev -r requirements_knowledge_graph.txt
#
# GPU Support (if available):
# pip install torch torchvision torchaudio --index-url https://download.pytorch.org/whl/cu118
#
# ===========================
# Environment Setup
# ===========================
#
# Virtual Environment:
# python -m venv food_kg_env
# source food_kg_env/bin/activate  # Linux/macOS
# food_kg_env\Scripts\activate     # Windows
# pip install -r requirements_knowledge_graph.txt
#
# Conda Environment:
# conda create -n food_kg python=3.9
# conda activate food_kg
# pip install -r requirements_knowledge_graph.txt
#
# ===========================
# Known Issues and Solutions
# ===========================
#
# 1. Windows C++ Compiler Issues:
#    Install Microsoft C++ Build Tools
#    OR use pre-compiled wheels: --find-links https://download.pytorch.org/whl/torch_stable.html
#
# 2. Memory Issues with Large Models:
#    Set environment variable: TRANSFORMERS_CACHE=/path/to/large/disk
#    Reduce batch sizes in configuration
#
# 3. Neo4j Connection Issues:
#    Ensure Neo4j server is running
#    Check firewall settings for ports 7474 and 7687
#
# 4. Redis Connection Issues:
#    Install Redis server: https://redis.io/download
#    Check default port 6379 availability
#
